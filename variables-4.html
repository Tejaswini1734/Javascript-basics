JavaScript Variable Scope

1. Global Scope

Explanation: Variables declared outside any function or block are globally scoped. They can be accessed anywhere in the code.


    var globalVar = "I'm a global variable";

    function accessGlobal() {
      console.log(globalVar); // ✅ Accessible here
    }

    accessGlobal();
    console.log(globalVar);   // ✅ Accessible here too

2. Function (Local) Scope

Explanation: Variables declared inside a function are local to that function and cannot be accessed outside it.


    function localScopeExample() {
      var localVar = "I'm a local variable";
      console.log(localVar); // ✅ Accessible here
    }

    localScopeExample();
    // console.log(localVar); // ❌ Error: localVar is not defined

3. Block Scope

Explanation: Variables declared with let and const inside blocks (like if, loops) are block-scoped.


    if (true) {
      let blockLet = "I'm block-scoped";
      const blockConst = "I'm also block-scoped";
      console.log(blockLet);   // ✅ Accessible inside block
      console.log(blockConst); // ✅ Accessible inside block
    }

    // console.log(blockLet);   // ❌ Error
    // console.log(blockConst); // ❌ Error

4. Variable Shadowing

Explanation: Shadowing occurs when a variable in a local or block scope has the same name as one in the outer scope. The inner one "shadows" the outer.


    let message = "Outer";

    function shadowExample() {
      let message = "Inner"; // Shadows the outer message
      console.log("Inside function:", message); // Outputs: Inner
    }

    shadowExample();
    console.log("Outside function:", message); // Outputs: Outer

⚠️ Common Issues from Misunderstanding Scope

  • Access Errors: Trying to access a variable outside its scope results in a ReferenceError.
  • Global Pollution: Declaring variables without let, const, or var can accidentally create global variables.
  • Shadowing Confusion: It's hard to track which variable is being used if names are reused in different scopes.
  • Using var: var ignores block scope, which may cause unexpected behavior.
